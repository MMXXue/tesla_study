## 🛠 代码提交规范 (Commit Message Convention)

为了保持项目历史的可读性和专业性，本项目遵循以下提交规范：

| 关键词 | 含义 | 使用场景 |
| :--- | :--- | :--- |
| **feat** | 新功能 (Feature) | 完成了 Day 4 的记忆功能、增加了新接口等 |
| **fix** | 修复 (Bug Fix) | 修复了连接中断、逻辑报错等 |
| **docs** | 文档 (Documentation) | 修改 README、添加代码注释等 |
| **style** | 格式 (Style) | 清理 `__pycache__`、调整代码缩进（不影响逻辑） |
| **refactor** | 重构 (Refactor) | 优化代码结构、移除冗余的 `venv` 等 |
| **chore** | 杂务 (Chore) | 更新 `.gitignore`、修改配置文件等 |

**格式示例：**
`feat: 增加用户 Session 存储逻辑`
