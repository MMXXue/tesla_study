# 性能排查清单（TPS & 延迟）

## 1. 启动一致性
- 固定启动命令，避免不同运行方式导致数据不可比。
- 压测前确认 `8000` 端口只有一个目标服务在监听。
- 关闭无关后台任务，减少噪声。

## 2. 压测方法
- 先扫并发，再选最优并发，不要直接拉满。
- 推荐使用 `perf_benchmark.py` 做基准：
  - `python backend/week2/perf_benchmark.py`
- 关注 `p95/p99`，不要只看平均值。

## 3. 关键指标阈值（建议）
- 成功率：`100%`
- 平均延迟：`< 50ms`
- p95 延迟：`< 120ms`
- 若出现 `PoolTimeout`，优先降低并发或增大客户端连接池。

## 4. 服务端热点
- API 热路径尽量只做入队，不做重逻辑。
- 减少每请求 JSON 反序列化/校验开销（高吞吐模式）。
- 使用 `uvloop + httptools`。
- 减少请求级日志输出。

## 5. Redis / Worker
- 检查 Redis 连接池是否打满。
- Worker 尽量批量消费，降低网络往返。
- 落库慢时会导致队列积压，需监控队列长度。

## 6. 调参顺序
1) 固定启动参数
2) 扫并发（10/20/30/40/50）
3) 找“TPS最高且延迟可接受”的区间
4) 再细调连接池、超时和 worker
